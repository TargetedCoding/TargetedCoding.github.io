Module 1: The Basic Building Blocks

Objective: To understand how to store and use data, and the core concept of immutability.

Now that your workshop is set up, it's time to learn how to handle the materials. In this module, we'll learn how to store information in labeled boxes called variables.

Step 1: Labeled Boxes (Variables)

In programming, we store data in variables. A key feature of Rust is that variables are immutable by default. This is a safety feature!

Analogy: Think of a variable as a labeled box. When you write let tasks_total = 5;, you are putting the number 5 into a box labeled 'tasks_total'. By default, Rust seals this box shut. You can look inside, but you can't change what's in it.

// This is an immutable variable.
let tasks_total = 5;

The mut Keyword

What if you need a box you can change? You must explicitly tell Rust by using the mut keyword (short for mutable). This is like putting a special "changeable" sticker on the box.

// This is a mutable variable.
let mut tasks_completed = 0;
println!("Tasks completed: {}", tasks_completed);

// We can change it because it's marked as `mut`.
tasks_completed = tasks_completed + 1;
println!("Tasks completed: {}", tasks_completed);

This "immutable by default" rule helps prevent a whole class of bugs where data gets changed unexpectedly.

Step 2: Different Kinds of Treasure (Scalar Types)

Variables can hold different kinds of data. Rust needs to know the type of data so it knows how to work with it. These basic types are called scalar types.

  • Integers: Whole numbers. Types include i32 (a 32-bit signed integer, meaning it can be negative) and u32 (a 32-bit unsigned integer, only positive). The number tells you the size. For now, i32 is a great default.
  • Floating-Point Numbers: Numbers with a decimal point, like f64.
  • Booleans: Can only be true or false. Used for making decisions.

Let's Build Our Todo App (Part 1)

It's time to apply what we've learned to our project. Let's start building our actual todo list. For now, we will just store a single task as text.

Open your src/main.rs file from Module 0 and replace the code with this:

fn main() {
    let mut todo_list = String::from("1. Learn Rust");
    println!("My tasks:\n{}", todo_list);

    // Let's add another task!
    todo_list.push_str("\n2. Understand variables");
    
    println!("\nMy tasks are now:\n{}", todo_list);
}

New Concepts Introduced:

  • String::from("..."): We introduce a new type, the String. While characters in single quotes are simple, text that can grow and change needs this more powerful type. String::from() creates a new, flexible string.
  • .push_str(): This is a "method" that belongs to the String type. It allows us to append more text to the end of our string.

Run your program again with cargo run. You'll see your updated task list! You have successfully used a mutable variable to modify data.
