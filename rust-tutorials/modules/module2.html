Module 2: Making Choices & Repeating Actions

Objective: To learn how to control the flow of the program with conditions and loops.

A program that only runs from top to bottom isn't very smart. To build truly useful things, our code needs to make decisions and perform repetitive tasks. This is called control flow.

Step 1: Forks in the Road (if/else)

To make decisions, we use if expressions. You can think of this like a fork in the road for your program.

Analogy: If a condition is true, your program goes down one path. If it's false, it can go down another path (using else) or simply continue on.

let number_of_tasks = 5;

if number_of_tasks > 0 {
    println!("You have tasks to do!");
} else {
    println!("You are all caught up!");
}

The condition (number_of_tasks > 0) must evaluate to a boolean: true or false.

Step 2: Repeating Actions (Loops)

Loops allow us to repeat a block of code multiple times. Rust has a few types of loops, but the most common and useful one is the for loop.

The for Loop

A for loop is perfect for executing a piece of code a specific number of times or for each item in a collection.

// The `1..=5` creates a range that includes 1, 2, 3, 4, and 5.
for number in 1..=5 {
    println!("This is task number {}", number);
}

The other loop types, loop (which runs forever until you break) and while (which runs as long as a condition is true), are also useful but the for loop is what you'll use most often.

Let's Build Our Todo App (Part 2)

Our current app is a bit limitedâ€”it can only store tasks in one long string. Let's upgrade it to use a Vector, which is a growable list that can hold multiple, separate items.

Replace the code in your src/main.rs file with this:

fn main() {
    // Create an empty, mutable vector to hold our `String` tasks.
    let mut tasks = Vec::new(); 

    // Add two tasks to the list.
    tasks.push("1. Learn Rust".to_string());
    tasks.push("2. Build a Todo App".to_string());
    tasks.push("3. Master control flow".to_string());

    println!("--- My Todo List ---");

    // Use a for loop to iterate over the tasks and print each one.
    for task in &tasks {
        println!("- {}", task);
    }
}

New Concepts Introduced:

  • Vec::new(): This creates a new, empty vector. We make it mut so we can add things to it.
  • .push(): This is a method that adds a new element to the end of the vector.
  • .to_string(): We need to convert our simple text (a &str) into the more flexible String type that our vector can own. We'll explore this more later!
  • for task in &tasks: This is the magic! The for loop iterates through our tasks vector. In each iteration, the variable task holds one of the items from the list.

A Sneak Peek: Notice that little & symbol before tasks? That's a very important part of Rust called a "borrow." It lets our loop "look at" the items in the vector without taking them. We will dive deep into this superpower in the next modules!

Run cargo run now. Your output will be a neatly formatted list of your tasks!
